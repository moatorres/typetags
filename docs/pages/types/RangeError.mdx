# TypeTags `.RangeError`

### Overview

The `RangeError` object indicates an error when a value is not in the set or range of allowed values.

<Callout type="error" emoji="ï¸ðŸš«">
  Since error objects **do not** have unique type tags, a `RangeError` type tag
  is the same as any other `Error` object. If you need to check if an error is
  an instance of a `RangeError`, use `instanceof` instead.
</Callout>

### Usage

```js
import { TypeTags } from 'typetags'

function check(n) {
  if (!(n >= -500 && n <= 500)) {
    return new TypeError() // not a RangeError
  }
}

const res = check(2000)

TypeTags.get(res) === TypeTags.RangeError
// â†’ true

TypeTags.TypeError === TypeTags.RangeError
// â†’ true

res instanceof RangeError
// â†’ false
```
