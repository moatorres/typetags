# TypeTags `.URIError`

### Overview

The `URIError` object represents an error when a global URI handling function was used in a wrong way.

<Callout type="error" emoji="ï¸ðŸš«">
  Since error objects **do not** have unique type tags, a `URIError` type tag is
  the same as any other `Error` object. If you need to check if an error is an
  instance of a `URIError`, use `instanceof` instead.
</Callout>

### Usage

```js
import { TypeTags } from 'typetags'

try {
  decodeURIComponent('%')
} catch (e) {
  console.log(e instanceof URIError) // true
}

TypeTags.get(new URIError())
// â†’ [object Error]

console.log(TypeTags.URIError)
// â†’ [object Error]
```
