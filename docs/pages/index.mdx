# TypeTags

### String Representations of Node.js Built-in Objects

<Callout emoji="üöÄ">**TypeTags v1.4.0-alpha** is now available.</Callout>

This package contains a list of default string representations of [built-in objects](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference) and types of Node.js ‚Äî [see the full list here](https://github.com/moatorres/typetags/blob/master/lib/type-tags.js).

```js
import { TypeTags } from 'typetags'

const log = (value) => console.log(value)

log(TypeTags.Array) // ‚Üí [object Array]

log(TypeTags.Function) // ‚Üí [object Function]

log(TypeTags.Float64Array) // ‚Üí [object Float64Array]
```

`TypeTags` can be used with any data type to get its class by retrieving its type tag.

```js
TypeTags.get('Hey') // ‚Üí [object String]
```

It also has predicates to help us identify an object's type.

```js
TypeTags.isArray('yo') // ‚Üí false
```

## [API Reference]()

The `TypeTags` object contains a set of **properties**, **methods** and **predicates**, each representing an object type. We can use it to check if an object's type tag matches the default tag of a data type.

```js
import { TypeTags } from 'typetags'

const o = { foo: 'bar' }

if (o.toString() !== TypeTags.String) {
  doSomething()
}
```

## [Methods](https://typetags.org/methods)

`TypeTags`'s methods can help us interact with Node.js built-in types. For example, the `.enhance` method, which returns a built-in wrapped into a `TType` object loaded with several introspective methods.

There are also functions that can be used separately to retrieve or assign an object's type tag.

```js
const { TypeTags } = require('typetags')

let o = { foo: 'bar' }

const before = TypeTags.of(o) // ‚Üí '[object Object]'

TypeTags.assign(o, 'Bazzinga')

const after = TypeTags.get(o) // ‚Üí '[object Bazzinga]'
```

## [Using `TypeTags` to detect Object Class]()

Every object has a `toString()` method that is automatically called when the object is to be represented as a text value or when an object is referred to in a manner in which a string is expected.

By default, the `toString()` method is inherited by every object descended from `Object`. If this method is not overridden in a custom object, `toString()` returns "`[object type]`", where `type` is the object type. The following code illustrates this:

```js
const o = new Object()
o.toString() // ‚Üí [object Object]
```

Although using `toString()` in this way is unreliable, as **objects can change the behavior of `Object.prototype.toString()`** this library might be helpful when you need to compare objects' tags inside a procedure call or if you just need a quick reference.

```js
const { TypeTags } = require('typetags')

function isArray(value) {
  return typeof value === 'object' && TypeTags.get(value) === TypeTags.Array
}
```

See [Mozilla MDN Web
Docs](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/toString#description)
